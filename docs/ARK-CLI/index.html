ARK-CLI

工具说明

ark-cli 是一个基于 Vue.js 进行快速开发的脚手架，为开发者提供初始化项目及复杂逻辑项目，
提供上传自己作品及下载其它组件的开发命令，使用者成为arksha 生态园的提供者，减少开发者重复无效工作、
避免无效沟通，提升工作效率

安装命令 -全局安装

npm install @syman/ark-cli -g
npm i webpack -g (-g全局安装,) npm i webpack-cli -g

配置项目

新建项目 ark init

  命令行  输入 name： 创建项目名称

init 命令执行后、启动项目

  cd 项目名称  
  npm run serve

发布项目命令

ark login   登录项目  命令行输入用户及密码
ark publish  ( 执行上传命令)

下载项目命令

ark install 组件名称@版本号

项目配置规则

  ark-cli  支持作者把自己的文件或者组件 上传到arkshare 平台,共享给开发者使用交流

   1. 依赖第三方插件的我们建议使用外链链接，请注意在使用import 和外链 的抛出的对象不同

   2. 注意打包文件项目中的图片的引用路径地址是否正确

   3. 每个包名的名称必须唯一，每次上传版本版本号必须递增

akrcli 上传配置

>    ark.build.config.json—— [上传多个文件到ark-cli时需要的配置文件]
    此文件需要提供当上传多个组件时，每个组件对应的包名（外层组件文件夹名称,应与每个组件的配置文件 build.config.json中的componentName保持一致）


      添加上传文件的 路径
      [
        {
            "path": "./src/pageDemo1"
        }

        ]

> src/pageDemo1/build.config 文件

    配置 上传文件的信息(包含文件的插件、文件标签、文件分类)


  此文件需提供所有组件用到的相关依赖，请提供资源的cdn链接，资源的包名以及版本号；
            {
              "componentName": "Test",             #指定包名，应与外层组件文件夹名称保持一致

              "version":"1.0.0"                    #指定版本号
              
              "componentPath": "/Test.vue",   #指定需要上传的组件的在当前目录下的绝对路径

              
              "classifyId":[""],   #分类名称（请按照文档填写）

               "tag":[""],   # 标签
               "coverImage":"",  # 封面图片链接
              "dependencies": [                //组件的相关依赖
                {
                  "packageName": "burgeon-ui",    //必须要提供的包名
                  "packageVersion": "1.0.21",     //必须要提供的包的版本号
                  "packageCDN": ""   //尽量提供的包的cdn链接
                },
                {
                  "packageName": "vue",       //必须要提供的包名
                  "packageVersion": "2.6.10", //必须要提供的包的版本号
                  "packageCDN": ""            //如果没有找到cdn链接，可以为空字符串
                },
                  "packageName": "vue-router",
                  "packageVersion": "3.0.2",
                  "packageCDN": ""
                }
              ]
            }
