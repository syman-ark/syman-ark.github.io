The Usage Of ARK CLI

安装命令

npm install @syman/ark-cli -g 全局安装

配置项目

新建项目 ark init

  命令行  输入 name： 创建项目名称

启动项目 cd 项目名称 npm run serve

  init 完成后启动项目

发布项目配置

  build.config.json ————————资源文件
  ```
  此文件需提供所有组件用到的相关依赖，请提供资源的cdn链接，资源的包名以及版本号；
            {
              "componentName": "Test",             #指定包名，应与外层组件文件夹名称保持一致
              "version":"1.0.0"                    #指定版本号
              "componentPath": "/Test.vue",   #指定需要上传的组件的在当前目录
              下的绝对路径
              "classifyId":[""],   #分类名称（请按照文档填写）
               "tag":[""],   # 标签
               "coverImage":"",  # 封面图片链接
              "dependencies": [                //组件的相关依赖
                {
                  "packageName": "burgeon-ui",    //必须要提供的包名
                  "packageVersion": "1.0.21",     //必须要提供的包的版本号
                  "packageCDN": "https://unpkg.com/burgeon-ui@1.0.20/dist/burgeon.min.js"   //尽量提供的包的cdn链接
                },
                {
                  "packageName": "vue",       //必须要提供的包名
                  "packageVersion": "2.6.10", //必须要提供的包的版本号
                  "packageCDN": ""            //如果没有找到cdn链接，可以为空字符串
                },
                  "packageName": "vue-router",
                  "packageVersion": "3.0.2",
                  "packageCDN": ""
                }
              ]
            }
  
         ```
         ark.build.config.json—— [上传多个文件到ark-cli时需要的配置文件]
             此文件需要提供当上传多个组件时，每个组件对应的包名（外层组件文件夹名称,应与每个组件的配置文件 build.config.json中的componentName保持一致）

            [
              {
                "path": "./src/component"
              }
            ]
         ```
         README 是操作文档
         UPDATELOG 更新日志

发布项目命令

ark login   登录项目  命令行输入用户及密码
ark publish  ( 执行上传命令)

下载项目命令

ark install 组件名称@版本号
